# パターン：Bloggerの制約を克服し、モダンなUXを動的に構築する

## 課題
Blogger（Google Blog）は基本設計が古く、現代的なブログ運営において以下の「壁」に直面する。

1.  **UXの欠如:** 目次、読了率表示、タイピング演出など、読者の離脱を防ぐリッチな機能が標準で用意されていない。
2.  **パフォーマンスの不備:** 画像のWebP変換配信や、動的なリサイズが自動化されておらず、表示速度が低下しやすい。
3.  **システム仕様の歪み:** 日本語として不自然な日付形式や、検索結果の表示件数が不安定になる仕様があり、デザインの整合性が保ちにくい。

## やったこと（実践内容）

### 1. 「後付け（JS注入）」によるUXの動的強化
既存のHTML（テンプレート）を複雑にせず、JavaScriptでDOMをスキャンして機能を後付けする。
- **目次生成:** 記事内の見出し（H2, H3）を抽出し、IDを自動付与してサイドバー等に目次を生成。スクロール連動（ScrollSpy）も実装。
- **読了率表示:** スクロール位置と記事の高さを計算し、上部にプログレスバーを表示。

### 2. URL・配信パラメータの動的書き換え
Bloggerのシステムが生成するURLを、フロントエンド側でリアルタイムに置換・強制する。
- **画像最適化:** 正規表現を用いて、画像URL内のサイズ指定を `/w750-rw/` 等に書き換え、WebP形式かつ最適幅での配信を強制。
- **検索制御:** 全てのラベルリンクに `max-results=7` を強制付与。これにより、どのリンクから流入しても表示件数を固定し、レイアウト崩れを防ぐ。

### 3. 表記とデータの正規化
不自然な表示データや静的なSVGを、スクリプトで補正・再描画する。
- **日付形式の統一:** 「月 27, 2026」のような不自然な順序を、正規表現で「YYYY/MM/DD」に組み替える。
- **動的SVG描画:** プレーンな数値データ（ddタグ等）をソースに、SVGの座標をリアルタイム計算してレーダーチャートを描画。

## なぜ効くのか（洞察）

- **テンプレート依存の最小化:** システムの根幹（XMLテンプレート）をいじりすぎないため、テーマ変更時もスクリプトの移植だけで機能を再現でき、メンテナンス性が高い。
- **UXの一貫性確保:** 表示件数の固定や日付の正規化により、どのページでもトーン＆マナーが統一され、サイトの信頼性が向上する。
- **リソースの最適化:** サーバー側が対応していない画像変換などをURL操作のみで実現するため、サーバー負荷を抑えつつ高速化が可能。

# パターン：Blogger APIを駆使した高度なページネーションの再構築

## 課題
Bloggerの標準ページネーションは「次の投稿」「前の投稿」という相対的な移動しかできず、以下の問題が発生する。

1.  **利便性の欠如:** ユーザーが任意のページ番号へ直接ジャンプできない。
2.  **表示件数の不安定さ:** アーカイブ（月別）や検索結果において、表示件数がシステム側の設定に依存し、デザイン（例：1ページ7件）を維持できない。
3.  **メタデータの不在:** フロントエンド側で「総記事数」や「総ページ数」を把握する標準的な手段がなく、動的なUI構築が困難。

## やったこと（実践内容）

### 1. フィードAPI（JSON）によるメタデータの取得
BloggerのJSONフィード（`/feeds/posts/summary`）を `max-results=0` で非同期取得（fetch）し、APIが返す `openSearch$totalResults` から正確な「総記事数」を算出する。
- 検索クエリ（q）が含まれる場合は、フィード内の `entry` 数で補完するロジックを実装し、精度の低い検索ヒット数表示をカバーする。

### 2. 動的なページ番号ナビゲーションの生成
取得した総記事数を、あらかじめ定義した `maxResults`（例：7件）で割り、ページ番号リンクを動的に描画する。
- **UI制御:** 現在のページを強調し、前後数ページ以外は `...` で省略するロジックにより、ページ数が多い場合でもUIが崩れないように設計。



### 3. `updated-max` を利用した正確なページ遷移
Bloggerの仕様（特定の投稿日時以前を表示する `updated-max` パラメータ）を利用して、指定ページへのジャンプを実現する。
- `window.pager` 関数内で、遷移先ページの先頭記事をAPIで特定。
- 取得した記事の投稿日時（published）を抽出し、正確なURLパラメータを組み立ててリダイレクトを行う。

### 4. アーカイブページにおける強制件数フィルタリング
月別アーカイブページにおいて、URLパラメータおよびDOM操作の二段階で表示件数を制御する。
- **URL制御:** `max-results` が付与されていない場合に自動で再ロード。
- **DOM制御:** `DOMContentLoaded` および `load` イベント時に、`max-results` を超える要素（`.post-outer`）を `display: none !important` で強制的に非表示化。

## なぜ効くのか（洞察）

- **「動的解決」による標準機能の完全上書き:** Bloggerサーバーサイドの制約（固定的なページネーション）を、フロントエンドでAPIを叩く「後付けの頭脳」によって補完・置換できる。
- **デザインの一貫性確保:** アーカイブや検索結果など、システム側が表示件数を動的に変えてしまうページにおいても、スクリプトがURLとDOMの両面から制御することで、タイルレイアウト等のグリッドデザインを死守できる。
- **データ駆動型のナビゲーション:** 静的なリンクではなく、その時々の正確な記事数に基づいたナビゲーションを提供できるため、記事の増減に自動で追従し、デッドリンクを発生させない。

  # パターン：Bloggerの外部連携とユーザー体験（UX）の高度化

## 課題
Blogger単体では解決が難しい、以下の高度な運用課題を解消する。
1.  **動的なデータ活用:** 人気記事ランキングや関連記事を、静的なテンプレートだけで実装するのは限界がある。
2.  **ユーザー行動の可視化:** Google Analytics以外に、読了率や滞在時間をスコア化して独自のランキングを作りたい。
3.  **表示パフォーマンスと広告の両立:** 自動広告によるレイアウト崩れを防ぎつつ、AdSenseを効率的に配信したい。
4.  **ウィジェットの操作性:** 標準のアーカイブウィジェットが使いにくく、特定の件数制限（7件）にも対応していない。

## やったこと（実践内容）

### 1. GAS（Google Apps Script）を活用した独自ログ・ランキングシステム
フロントエンドとGASを連携させ、サーバーレスな計測環境を構築。
- **記録係:** `visibilitychange` と `navigator.sendBeacon` を組み合わせ、離脱直前の滞在時間とスクロール率を計算。GASへデータを飛ばし、独自の「記事スコア」を算出する。
- **表示係:** GASから集計済みのJSONを取得し、アイコン付きのランキングテーブルとして描画。

### 2. JSON-in-scriptによる動的な関連記事生成
BloggerのフィードAPIを非同期で呼び出し、サイト内の最新記事からランダムに関連記事カードを生成。
- **最適化:** `media$thumbnail` のURLを正規表現で置換し、高画質なWebP画像（w600）を配信。
- **デザイン:** Font Awesomeを統合したカテゴリラベルとスニペット表示により、カード型のモダンなUIを提供。

### 3. アーカイブウィジェットの挙動カスタマイズ
標準のリスト形式アーカイブに、年別の展開・折り畳み機能と、詳細ページへの遷移ロジックを注入。
- **クリック判定:** クリック位置（X座標）によって「年別ページへの遷移」と「リストの展開」を使い分けるハイブリッドなUIを実現。
- **物理削除ロジック:** CSSによる非表示（`display: none`）だけでなく、`removeChild` を用いてDOMから8件目以降を物理的に削除し、ページ内の不整合を完全に解消。

### 4. 広告配信の最適化とレイアウト保護
- **自動広告の排除:** `MutationObserver` を用い、Googleが自動で挿入する広告ユニットを検知して即座に削除。意図しないデザイン崩れを徹底的に防ぐ。
- **遅延読み込み (Lazy Load):** AdSenseのスクリプト本体を `window.load` の2秒後に動的挿入。ページの初期読み込み速度（LCP）への影響を最小限に抑える。

## なぜ効くのか（洞察）

- **「ゼロ・サーバー」での高度な機能実現:** GASをバックエンドとして活用することで、サーバー維持費をかけずにデータベース的な機能（ランキング等）を実装できる。
- **UXの「静寂」を守る:** 自動広告の強制削除や遅延読み込みにより、ユーザーにストレスを与えないクリーンな閲覧環境を維持しつつ、収益化の最適化ポイントを自社でコントロールできる。
- **情報の構造化と提示:** 関連記事のランダム表示やアーカイブの階層制御により、過去記事への回遊性を高め、サイト全体のPV底上げに直結する。
